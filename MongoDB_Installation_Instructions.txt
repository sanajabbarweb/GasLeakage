
### MongoDB Installation Steps for Ubuntu 23.10 (Using Ubuntu 22.04 Repository as a Workaround)

If you encounter issues while installing MongoDB on Ubuntu 23.10, these steps guide you through using the repository for Ubuntu 22.04 (codename: `jammy`), as MongoDB may not have a `mantic` repository.

---

#### 1. Remove Existing MongoDB Repository (If Any)

```bash
sudo rm /etc/apt/sources.list.d/mongodb-org-6.0.list
```

#### 2. Add MongoDB GPG Key

Add MongoDBâ€™s GPG key to your system.

```bash
curl -fsSL https://pgp.mongodb.com/server-6.0.asc | sudo gpg -o /usr/share/keyrings/mongodb-server-6.0.gpg --dearmor
```

#### 3. Add the Ubuntu 22.04 (Jammy) MongoDB Repository

Use the following command to add the MongoDB repository for `jammy`:

```bash
echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-6.0.gpg ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/6.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-6.0.list
```

#### 4. Update Package List

Refresh the package list so that it includes MongoDB from the new repository:

```bash
sudo apt update
```

#### 5. Install MongoDB

Attempt to install MongoDB:

```bash
sudo apt install -y mongodb-org
```

---

#### Additional Troubleshooting Steps

1. **Verify Repository Availability**:

   Use the following command to verify if the repository is correctly recognized:

   ```bash
   apt-cache policy mongodb-org
   ```

   This command should display available versions of `mongodb-org` if the repository setup is correct.

2. **Alternative Installation via Snap**:

   If issues persist, try installing MongoDB via Snap.

   ```bash
   sudo snap install mongodb
   ```

   This approach installs MongoDB using the Snap package manager.

---

These steps should help you successfully install MongoDB on Ubuntu 23.10 by leveraging the `jammy` repository.
